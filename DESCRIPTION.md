# AI Agent System: Алгоритмы и архитектура (DDD Edition)

**AI Agent System** — это мульти-агентная система на Laravel, реализованная с использованием принципов Domain-Driven Design (DDD) и реактивного event-driven подхода для оркестрации задач.

---

## 1. Архитектура событийного цикла

Система работает как бесконечный цикл "Планирование -> Выполнение -> Анализ", пока не будет достигнут финальный результат.

### Основные события (Workflow):
События находятся в пространстве имен `App\Application\Ai\Conversation\Event`:

- **`WorkflowStarted`**: Инициализация процесса.
- **`StepRequested`**: Запрос к планировщику на определение следующего шага.
- **`StepPlanned`**: Шаг определен и готов к выполнению (содержит объект `PlanStep`).
- **`StepExecuting`**: Агент начал выполнение задачи.
- **`StepCompleted`**: Агент завершил работу, результат получен и добавлен в историю.
- **`WorkflowCompleted`**: Финальная точка, процесс завершен.

## 2. Ключевые алгоритмы системы

Ниже приведено подробное описание того, как именно обрабатывается ваш запрос.

### А. Алгоритм оркестрации (Жизненный цикл запроса)
Этот алгоритм управляет переходами между состояниями системы.

1.  **Запуск**: 
    - Класс `StartConversationUseCase` (метод `execute`) создает объект `Conversation`.
    - Генерирует событие `WorkflowStarted` (логирование) и `StepRequested` (запуск цикла).
2.  **Планирование**:
    - `PlanNextStepListener` получает запрос и передает его в `PlanNextStepUseCase`.
    - Используется **LLM-планировщик** (`LlmDynamicPlanner::decideNextStep`), который на основе истории беседы решает: отправить задачу исследователю (`research`), подвести итог (`summary`) или закончить работу (`null`).
    - Если шаг определен, создается `PlanStep` и генерируется событие `StepPlanned`.
3.  **Выполнение**:
    - `ExecuteStepListener` ловит `StepPlanned` и через `ExecuteStepUseCase` вызывает `AgentRegistryExecutor`.
    - Система выбирает нужного агента и запускает его метод `execute`.
    - По завершении генерируется `StepCompleted`.
4.  **Анализ и Зацикливание**:
    - `UpdateStateListener` передает результат в `ProcessStepResultUseCase`.
    - Здесь создается **новая копия** объекта `Conversation` (так как он иммутабелен) с добавленной записью в историю (`withStepResult`).
    - Если ответ найден (метка `[RESEARCH_FINISHED]`) или достигнут лимит шагов (10), генерируется `WorkflowCompleted`.
    - Иначе генерируется новый `StepRequested`, и мы возвращаемся к пункту 2.

### Б. Алгоритм работы ResearchAgent (Исследователь)
Агент `ResearchAgent` не просто делает один запрос к ИИ, он работает итеративно (в цикле).

1.  **Исполнение задачи** (`execute`):
    - Делает запрос к LLM (метод `prompt`).
    - LLM может вызвать доступные инструменты (Tools), например, поиск в базе.
2.  **Рефлексия (Reflection)**:
    - Метод `reflect` просит LLM проанализировать полученный результат.
    - Если LLM видит, что ответ найден, она возвращает `"finish": true`.
    - Если данных не хватает, она формулирует `"next_task"` — уточненный запрос для следующей итерации.
3.  **Завершение**: Если за 3 итерации ответ не найден, агент возвращает то, что успел собрать.

### В. Алгоритм RAG (Поиск по базе знаний)
Реализован в инструменте `VectorSearchTool` (метод `handle`).

1.  **Перефразирование**: `LlmQueryRewriter::rewrite` преобразует вопрос пользователя в оптимальный поисковый запрос для векторной базы.
2.  **Поиск**: `EloquentDocumentRepository::search` выполняет поиск похожих фрагментов текста с помощью расширения `pgvector`.
3.  **Переранжирование (Reranking)**: `LlmReranker::rerank` отправляет найденные фрагменты в LLM, чтобы она выбрала 5 самых релевантных. Это отсеивает "шум" и повышает точность.

## 3. Основные компоненты (Классы)

### Domain Layer (Бизнес-правила)
- **`Conversation`** (`App\Domain\Ai\Conversation\Conversation`): Главный объект, хранящий историю и состояние. Методы `start`, `withStepResult`, `isFinished`.
- **`PlanStep`**: Описание шага (какой агент и что делает).

### Application Layer (Оркестрация)
- **`StartConversationUseCase`**: Точка входа для новых запросов.
- **`ProcessStepResultUseCase`**: Логика принятия решения о продолжении цикла или его завершении.
- **Слушатели (`Listeners\*`)**: "Клей" системы, передающий управление между этапами через события.

### Infrastructure Layer (Техника)
- **`LlmDynamicPlanner`**: Мозг системы, решающий, что делать дальше.
- **`ResearchAgent`**: Сложенный агент с логикой итераций и рефлексии.
- **`VectorSearchTool`**: Инструмент, объединяющий поиск, перефразирование и переранжирование.

## 4. Запуск и Тестирование

### Консоль
```bash
docker-compose exec app php artisan ai:test "Твой запрос"
```

### Веб-интерфейс
`AiController` предоставляет эндпоинты для работы с фронтендом, включая `stream` для отображения хода мыслей агентов в реальном времени.

### Тесты
```bash
docker-compose exec app php artisan test
```
