# AI Agent System: Реактивная событийная модель

**AI Agent System** — это мульти-агентная система на Laravel, использующая реактивный event-driven подход для оркестрации задач.

---

## 1. Архитектура событийного цикла

Система управляется событиями. Это обеспечивает гибкость, возможность асинхронного выполнения и лучшую масштабируемость.

### Основные события (Workflow):
- **`App\Ai\Events\Workflow\WorkflowStarted`**: Инициализация процесса.
- **`App\Ai\Events\Workflow\StepRequested`**: Запрос к планировщику на определение следующего шага.
- **`App\Ai\Events\Workflow\StepPlanned`**: Шаг определен и готов к выполнению.
- **`App\Ai\Events\Workflow\StepExecuting`**: Агент начал выполнение задачи.
- **`App\Ai\Events\Workflow\StepCompleted`**: Агент завершил работу, результат получен.
- **`App\Ai\Events\Workflow\WorkflowCompleted`**: Финальная точка, процесс завершен.

---

## 2. Жизненный цикл запроса (Request Cycle)

Процесс обработки запроса представляет собой реактивную цепочку событий:

1.  **Инициализация (`Supervisor`)**:
    - Пользователь отправляет сообщение.
    - `Supervisor` создает начальный `AgentState`.
    - Генерируются события `WorkflowStarted` и `StepRequested`.

2.  **Планирование (`PlanNextStepListener`)**:
    - Слушает `StepRequested`.
    - Вызывает `DynamicPlanner::decideNextStep()`.
    - Если планировщик возвращает новый `Step`, генерируется `StepPlanned`.
    - Если планировщик возвращает `null`, генерируется `WorkflowCompleted`.

3.  **Выполнение (`ExecuteStepListener`)**:
    - Слушает `StepPlanned`.
    - Генерирует `StepExecuting` (для уведомления UI/логов).
    - Запрашивает агента из `AgentRegistry`.
    - Вызывает `$agent->execute()`.
    - По завершении (успешном или нет) генерирует `StepCompleted`.

4.  **Обновление состояния (`UpdateStateListener`)**:
    - Слушает `StepCompleted`.
    - Создает новый экземпляр `AgentState` с добавленным результатом в историю (метод `withStepResult`).
    - Проверяет `state->isFinished()`:
        - Если процесс завершен (например, выполнен `summary`), генерирует `WorkflowCompleted`.
        - Если нет, генерирует новый `StepRequested`, замыкая цикл.

5.  **Завершение (`WorkflowCompleted`)**:
    - Сигнализирует о конце обработки.
    - В потоковом режиме (SSE) отправляет клиенту финальный результат и метку `[DONE]`.

## 3. Основные компоненты

### Supervisor (Супервизор)
Точка входа. Создает начальное состояние `AgentState` и запускает цепочку событий.
- **Класс**: `App\Ai\Core\Supervisor`
- **Метод `handle(string $input)`**: Инициализирует состояние и вызывает `WorkflowStarted` и `StepRequested`.

### Dynamic Planner (Планировщик)
Принимает решение о следующем шаге на основе истории и текущего состояния.
- **Класс**: `App\Ai\Core\DynamicPlanner`
- **Интерфейс**: `App\Ai\Core\Interfaces\DynamicPlannerInterface`
- **Метод `decideNextStep(AgentState $state)`**: Возвращает объект `Step` или `null` для завершения.

### Agent Registry (Реестр агентов)
Карта доступных агентов, используемая исполнителем для получения нужного экземпляра.
- **Класс**: `App\Ai\Core\AgentRegistry`
- **Регистрация**: В `AppServiceProvider` через `singleton`.

### Listeners (Слушатели)
Реализуют логику перехода между состояниями (пространство имен `App\Ai\Listeners\Workflow`):
1. **`PlanNextStepListener`**: Слушает `StepRequested`, вызывает планировщик. В зависимости от результата вызывает `StepPlanned` или `WorkflowCompleted`.
2. **`ExecuteStepListener`**: Слушает `StepPlanned`, вызывает `StepExecuting`, находит агента в реестре и запускает его метод `execute()`. По завершении вызывает `StepCompleted`.
3. **`UpdateStateListener`**: Слушает `StepCompleted`, обновляет историю в `AgentState` (через `withStepResult`) и решает: запустить новый `StepRequested` или завершить workflow через `WorkflowCompleted`.

---

## 3. Агенты

### ResearchAgent (Исследователь)
Выполняет сбор данных. Содержит внутренний цикл (до 3 итераций) с механизмом рефлексии для уточнения поиска.
- **Класс**: `App\Ai\Agents\ResearchAgent`
- **Метод `execute(string $task, AgentState $state)`**: Выполняет поиск через инструменты и анализирует достаточность данных.

### SummaryAgent (Ответчик)
Синтезирует финальный ответ на основе всей накопленной истории исследований.
- **Класс**: `App\Ai\Agents\SummaryAgent`

### Reranker (Реранкер)
Обеспечивает переранжирование результатов поиска для повышения релевантности.
- **Класс**: `App\Ai\Core\Reranker`
- **Логика работы**:
    - Передает агенту `SmartAnonymousAgent` список ID и содержание найденных фрагментов.
    - Агент возвращает список ID наиболее релевантных фрагментов (макс. N).
    - Система парсит IDs из текстового ответа с помощью регулярных выражений.
    - **Фолбэк**: Если парсинг не удался, возвращаются первые N фрагментов.
    - Результаты возвращаются в порядке релевантности, определенном агентом.

---

## 4. Состояние (AgentState)
Объект состояния является иммутабельным. Каждое обновление (через `withStepResult`) создает новый экземпляр состояния с обновленной историей и очищенным текущим шагом.
- **Класс**: `App\Ai\Core\State\AgentState`
- **Метод `isFinished()`**: Определяет конец процесса (например, после успешного выполнения `summary`).

---

## 5. Запуск
Для тестирования системы используется консольная команда:
```bash
php artisan ai:test "Твой запрос"
```
Команда запускает `Supervisor`, который инициирует реактивную цепочку событий. Прогресс выполнения логируется и отображается в консоли.

Для веб-интерфейса реализован `AiController`, поддерживающий:
- **`POST /chat`**: Синхронный запуск (возвращает статус старта).
- **`GET /stream`**: Потоковая передача событий (SSE) для отображения прогресса в реальном времени.
