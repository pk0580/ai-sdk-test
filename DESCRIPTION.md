# Описание реализации AI Agent System

## Шаг 1: Event-driven ядро

В качестве первого шага реализована инфраструктура событий (Laravel Events) для управления жизненным циклом агента.

### Почему события?
Использование событий позволяет:
- Избавиться от жестко закодированных циклов `while`.
- Сделать систему расширяемой (можно добавить логирование, уведомления или новых агентов без изменения ядра).
- Реализовать асинхронную обработку шагов через очереди.

### Основные события:
- `UserMessageReceived`: Входная точка, сообщение от пользователя.
- `PlanCreated`: Создан план действий (массив шагов).
- `ToolCalled`: Вызван конкретный инструмент (Tool).
- `ToolResultReceived`: Получен результат работы инструмента.
- `ReflectionGenerated`: Агент проанализировал результат и решил, что делать дальше.
- `StepCompleted`: Текущий шаг плана завершен.

## Шаг 2: Planner (LLM)

На этом этапе реализован мозг агента — Планировщик (`Planner`), который использует LLM (ИИ) для превращения запроса пользователя в конкретный план действий.

### Что делает Планировщик?
1. **Анализирует запрос**: "Сколько будет 25 * 17?"
2. **Выбирает инструменты**: Для расчетов нужен `calculator`, для поиска знаний — `vector_search`.
3. **Формирует шаги**: Создает последовательность действий в формате JSON, понятном для программного выполнения.

### Почему это важно?
- **Экономия ресурсов**: Агент сразу видит путь к решению, что уменьшает лишние обращения к ИИ.
- **Предсказуемость**: Мы получаем структурированный список шагов (DTO `Plan` и `Step`) вместо нечеткого текста.
- **Обработка ошибок**: Если ИИ ошибается или не отвечает, предусмотрен "запасной" план (fallback), чтобы система не зависла.

### Технические детали
1. **Класс Planner**:
   - Использует `Laravel\AI\Facades\AI` для общения с языковыми моделями.
   - Имеет жестко заданный системный промпт с описанием доступных инструментов (`calculator`, `vector_search`).
   - Настроен на возврат данных в формате JSON (`->json()`).

2. **Интеграция**:
   - `PlanListener` теперь не создает заглушку, а вызывает `Planner::generate($message)`.
   - Результат упаковывается в `PlanCreated` и передается дальше по цепочке событий.

### Технические детали (инфраструктура)
1. **Docker Окружение**:
   - Приложение работает на PHP 8.4-cli (официальный образ).
   - Основные сервисы: `app` (PHP), `db` (PostgreSQL + pgvector), `redis`, `worker`, `horizon`, `scheduler`, `ollama`.
   - Запуск тестов в докере: `docker exec ai-sdk-test-app-1 php artisan test`.

2. **Event-driven Цикл**:
   - Команда `ai:test` инициирует цепочку: `UserMessageReceived` -> `PlanCreated` -> `ToolCalled` -> `ToolResultReceived` -> `ReflectionGenerated` -> `StepCompleted` / `Finish`.
   - Текущая реализация эмулирует выполнение инструментов и принятие решений (Reflection), логируя каждый этап в `laravel.log`.

3. **Логирование**:
   - Все события логируются в `storage/logs/laravel.log`, что позволяет отслеживать "мысли" агента.

### Алгоритм работы Planner:
1. **Подготовка промпта**: Формируется системная инструкция (`INSTRUCTIONS`), которая строго определяет:
   - Роль ИИ (Планировщик).
   - Список доступных инструментов и их JSON-схемы параметров.
   - Обязательный формат ответа (валидный JSON с массивом `steps`).
2. **Запрос к LLM**:
   - Создается анонимный агент (`AnonymousAgent`) с инструкциями.
   - Текст пользователя передается в метод `prompt()`.
3. **Обработка ответа (Парсинг)**:
   - Полученный текст очищается от лишних символов (например, Markdown-остатков ` ```json `).
   - Извлекается содержимое между первой `{` и последней `}`.
4. **Валидация**:
   - Проверка через `json_decode`.
   - Проверка наличия обязательного поля `steps`.
5. **Fallback (Отказоустойчивость)**:
   - Если LLM вернула невалидный JSON, произошла ошибка сети или API, Планировщик не останавливает систему.
   - Создается "запасной" план с одним шагом (например, `vector_search` с общим запросом), чтобы агент мог продолжить работу.

### Архитектурные решения:
- Введены DTO (`Plan`, `Step`) для типизации данных, передаваемых между событиями.
- Структура папок соответствует предложенной в `STEPS.md` (размещение в `app/AI`).
