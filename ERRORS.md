# Ошибки и проблемы в директории .claude

В ходе анализа конфигурации Claude Code (.claude) были выявлены следующие несоответствия и потенциальные проблемы:

1. **Несоответствие структуры генератора и реального проекта:**
   - **Infrastructure:** Файл `generator.md` и шаблоны (`.stub`) ожидают структуру с Bounded Context, например: `App\Infrastructure\{Ctx}\Persistence\Eloquent\...`. В реальности в проекте используется плоская структура: `App\Infrastructure\Persistence\Eloquent\...`.
   - **Interface/Http:** Генератор ожидает контроллеры по пути `App\Interface\Http\{Ctx}\Controller`, но в проекте они находятся в `App\Interface\Http\Controller` (без разделения на контексты).
   - Это приведет к тому, что при генерации новых модулей Claude создаст структуру, которая не совпадает с текущей логикой проекта, нарушая единообразие.

2. **Проблема с `settings.local.json`:**
   - Файл `.claude/settings.local.json` отслеживается Git (`git ls-files` подтверждает наличие). Несмотря на то, что он добавлен в `.gitignore`, он был закоммичен в репозиторий.
   - Согласно `CLAUDE_INSTRUCTIONS.md`, этот файл должен быть строго локальным. Наличие его в репозитории с конкретным именем контейнера (`ai-sdk-test-app-1`) может вызывать конфликты у разных разработчиков.

3. **Уточнение формата путей (выполнено):**
   - В документации (`CLAUDE_INSTRUCTIONS.md`, `.claude/README.md`) теперь явно указано, что `CLAUDE_SRC_PREFIX` должен передаваться в формате Unix/WSL (например, `/mnt/c/...`), чтобы избежать проблем при сопоставлении путей в `php-postwrite.sh`.

4. **Несоответствие шаблона ServiceProvider:**
   - Шаблон `service-provider.stub` пытается зарегистрировать политики из неймспейса `App\Interface\Http\{{BoundedContext}}\Policy`, однако в текущей структуре проекта папка `Policy` отсутствует, а контроллеры и запросы не разделены по контекстам в слое Interface.

5. **Инструкции по установке:**
   - В `CLAUDE_INSTRUCTIONS.md` (линия 20) сказано "Создай .claude/settings.local.json", но файл уже существует в репозитории. Это сбивает с толку при первоначальной настройке.

**Рекомендации:**
- Обновить `generator.md` и стабы, чтобы они соответствовали фактической структуре проекта (или привести проект к структуре, описанной в правилах).
- Удалить `settings.local.json` из индекса Git (`git rm --cached`).
