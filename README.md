# AI Agent System

Современная мульти-агентная система на базе Laravel и AI SDK. Проект демонстрирует реализацию сложных ИИ-сценариев через цепочку специализированных агентов, инструментов и механизмов рефлексии.

## 🚀 Основные возможности

- **Multi-Agent Orchestration**: Динамическое планирование и выполнение задач с помощью нескольких специализированных агентов.
- **Hybrid Planning**: Сочетание детерминированных правил и гибкого планирования с помощью LLM (Large Language Models).
- **RAG (Retrieval-Augmented Generation)**: Поиск по базе знаний с использованием векторного хранилища (PostgreSQL + `pgvector`).
- **Iterative Reasoning (Loop)**: Агенты способны работать циклично, проверяя и уточняя результаты своих действий (Reflection).
- **Real-time Streaming**: Отображение "хода мыслей" агентов в реальном времени через Server-Sent Events (SSE).
- **Tool Integration**: Поддержка внешних инструментов, таких как калькулятор и векторный поиск.

## 🧠 Архитектура системы

Система построена на событийно-ориентированной архитектуре (Event-Driven):

1.  **Supervisor**: Точка входа, координирует работу всей системы.
2.  **Hybrid Planner**: Формирует план выполнения задачи (статический или на базе LLM).
3.  **Orchestration Executor**: Пошагово выполняет план, передавая контекст (`AgentState`) между агентами.
4.  **Агенты**:
    - **ResearchAgent**: Исследователь, ищет информацию и выполняет расчеты.
    - **SummaryAgent**: Саммаризатор, структурирует данные и готовит итоговый ответ.

## 🛠 Технологический стек

- **Backend**: Laravel 11, PHP 8.4
- **Database**: PostgreSQL + `pgvector`
- **AI Infrastructure**: Ollama (модели `llama3.1`, `nomic-embed-text`)
- **Queue & Events**: Redis, Laravel Horizon
- **Tools**: Prism-PHP (AI SDK)

## 🏗 Быстрый старт

Проект полностью контейнеризирован с помощью Docker.

1.  **Клонируйте репозиторий**:
    ```bash
    git clone <repository-url>
    cd ai-sdk-test
    ```

2.  **Запустите инфраструктуру**:
    ```bash
    docker-compose up -d
    ```
    *При первом запуске Docker автоматически скачает необходимые LLM-модели через Ollama.*

3.  **Настройте окружение**:
    Зайдите в контейнер и выполните стандартные команды Laravel:
    ```bash
    docker-compose exec app composer install
    docker-compose exec app php artisan migrate
    ```

4.  **Использование**:
    Откройте `http://localhost:8000` в браузере. Вы сможете протестировать систему в разных режимах (обычный запрос, стриминг, фоновое выполнение).

## 🔍 Инспекция и мониторинг

Для отладки и наблюдения за работой системы доступны следующие инструменты:

- **Laravel Horizon**: Мониторинг очередей и фоновых задач. Доступен по адресу `http://localhost:8000/horizon`. Здесь можно увидеть выполнение шагов оркестрации, которые были отправлены в очередь.
- **Adminer**: Легкий веб-интерфейс для работы с базой данных PostgreSQL. Доступен по адресу `http://localhost:8080`. Позволяет просматривать таблицу `vectors` и содержимое памяти агентов.
- **Reasoning Stream**: В веб-интерфейсе системы при использовании режима "Streaming" отображается детальный лог "размышлений" агентов (SSE события), включая вызовы инструментов и результаты рефлексии.
- **Логи контейнеров**: Для глубокой отладки используйте стандартную команду docker-compose:
  ```bash
  docker-compose logs -f app worker
  ```
